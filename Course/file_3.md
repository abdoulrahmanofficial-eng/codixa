# 🎯 أهداف الدرس

بعد الانتهاء من هذا الدرس ستكون قادرًا على:

- معرفة الفرق بين Hardware وSoftware.
- التعرف على أنواع البرمجيات.
- فهم العلاقة بين مكونات الجهاز والبرامج.
- معرفة ما يحتاجه أي برنامج حتى يعمل.

---

# مقدمة

أي جهاز كمبيوتر يتكون من جزأين أساسيين:

1. Hardware
2. Software

ولا يمكن لأي منهما العمل بدون الآخر.

---

# أولًا: ما هو Hardware؟

الـ Hardware هو كل جزء مادي يمكنك رؤيته أو لمسه.

أمثلة:

- الشاشة (Monitor)
- لوحة المفاتيح (Keyboard)
- الفأرة (Mouse)
- المعالج (CPU)
- الرامات (RAM)
- وحدة التخزين (SSD/HDD)
- اللوحة الأم (Motherboard)
- كارت الشاشة (GPU)

إذا استطعت لمسه فهو غالبًا Hardware.

---

# مثال من الحياة

تخيل أنك تمتلك سيارة.

الأجزاء التالية تعتبر Hardware:

- المحرك
- العجلات
- المقود
- الفرامل
- المقاعد

كلها أجزاء مادية.

---

# ثانيًا: ما هو Software؟

الـ Software هو مجموعة من البرامج والتعليمات التي تجعل الـ Hardware يعمل.

أمثلة:

- Windows
- macOS
- Linux
- Google Chrome
- Microsoft Word
- Visual Studio Code
- الألعاب
- تطبيقات الهاتف

هذه البرامج لا يمكن لمسها، لكنها تتحكم في الجهاز.

---

# مثال من الحياة

إذا كان الكمبيوتر عبارة عن جسم إنسان:

- Hardware = الجسم.
- Software = العقل والتعليمات.

الجسم وحده لا يفعل شيئًا.

والتعليمات وحدها لا يمكن تنفيذها بدون جسم.

---

# أنواع البرمجيات

تنقسم البرمجيات إلى نوعين رئيسيين:

## 1. System Software

وهي البرامج المسؤولة عن تشغيل الجهاز وإدارة مكوناته.

مثل:

- Windows
- Linux
- macOS
- Android
- iOS

هذه البرامج تسمى **أنظمة تشغيل**.

---

## ما هو نظام التشغيل؟

نظام التشغيل هو البرنامج الذي يدير الكمبيوتر بالكامل.

وظائفه:

- تشغيل البرامج.
- إدارة الملفات.
- إدارة الذاكرة.
- إدارة الأجهزة.
- التحكم في الطابعة والفأرة ولوحة المفاتيح.
- توفير واجهة للمستخدم.

بدون نظام تشغيل لن تستطيع استخدام الكمبيوتر بسهولة.

---

## 2. Application Software

وهي البرامج التي يستخدمها المستخدم لإنجاز مهام معينة.

مثل:

- Word
- Excel
- Chrome
- Photoshop
- WhatsApp
- Spotify
- VS Code

هذه البرامج تعمل فوق نظام التشغيل.

---

# العلاقة بينهما

```
المستخدم

↓

التطبيق (Application)

↓

نظام التشغيل (Operating System)

↓

الهاردوير (Hardware)
```

عندما تضغط على زر في برنامج، لا يتعامل البرنامج مع الشاشة أو المعالج مباشرة، بل يطلب من نظام التشغيل تنفيذ المهمة، ثم يقوم النظام بالتواصل مع العتاد.

---

# مثال عملي

عندما تضغط على زر حفظ (Save) في برنامج Word:

1. البرنامج يطلب حفظ الملف.
2. نظام التشغيل يستقبل الطلب.
3. نظام التشغيل يرسل البيانات إلى وحدة التخزين.
4. يتم حفظ الملف على القرص.

---

# هل يمكن تشغيل برنامج بدون نظام تشغيل؟

في أغلب الحالات: لا.

فالبرامج تعتمد على خدمات يوفرها نظام التشغيل، مثل:

- إدارة الذاكرة.
- عرض النوافذ.
- قراءة الملفات.
- استخدام الإنترنت.

---

# لماذا توجد أنظمة تشغيل مختلفة؟

لكل نظام مميزاته واستخداماته.

| النظام | الاستخدام الشائع |
|---------|------------------|
| Windows | الاستخدام العام والألعاب |
| macOS | أجهزة Apple والتصميم |
| Linux | الخوادم والبرمجة |
| Android | الهواتف الذكية |
| iOS | هواتف Apple |

---

# كيف تتواصل البرامج مع الهاردوير؟

مثال عند تشغيل أغنية:

```
تشغيل الأغنية

↓

برنامج الموسيقى

↓

نظام التشغيل

↓

كارت الصوت

↓

السماعات

↓

تسمع الصوت
```

---

# هل البرنامج يعرف مكان الملفات بنفسه؟

لا.

هو يطلب من نظام التشغيل الوصول إلى الملف، ثم يقوم النظام بإحضاره من وحدة التخزين.

---

# معلومة مهمة

يمكن تثبيت آلاف البرامج على جهازك، لكنها لا تعمل جميعًا في الوقت نفسه.

البرامج التي تفتحها فقط يتم تحميلها إلى RAM ليبدأ المعالج في تنفيذها.

---

# أخطاء شائعة

❌ الاعتقاد أن Windows هو الكمبيوتر.

الصحيح:

Windows مجرد نظام تشغيل يعمل على الكمبيوتر.

---

❌ الاعتقاد أن Chrome جزء من Windows.

الصحيح:

Chrome برنامج مستقل يعمل فوق نظام التشغيل.

---

❌ الاعتقاد أن حذف اختصار البرنامج من سطح المكتب يعني حذف البرنامج.

الصحيح:

قد يكون ما حُذف مجرد اختصار، بينما يظل البرنامج مثبتًا على الجهاز.

---

# ملخص الدرس

- Hardware هو كل ما يمكن لمسه.
- Software هو البرامج والتعليمات.
- System Software يشمل أنظمة التشغيل.
- Application Software يشمل البرامج التي يستخدمها المستخدم.
- البرامج تتواصل مع الهاردوير من خلال نظام التشغيل.

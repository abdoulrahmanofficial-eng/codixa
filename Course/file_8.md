# 🎯 أهداف الدرس

بعد الانتهاء من هذا الدرس ستكون قادرًا على:

- فهم معنى الخوارزمية (Algorithm).
- معرفة لماذا تعتبر الخوارزميات أساس البرمجة.
- كتابة خوارزميات بسيطة.
- تحويل أي مشكلة إلى خطوات مرتبة.
- التعرف على خصائص الخوارزمية الجيدة.

---

# مقدمة

قبل أن يكتب أي مبرمج محترف الكود، فإنه يفكر في **طريقة حل المشكلة**.

هذه الطريقة تسمى:

> **Algorithm (الخوارزمية)**

كل برنامج تستخدمه يوميًا يعتمد على آلاف الخوارزميات.

---

# ما هي الخوارزمية؟

الخوارزمية هي:

> **مجموعة من الخطوات المرتبة والمنطقية لحل مشكلة معينة أو تنفيذ مهمة محددة.**

ببساطة...

هي خطة العمل قبل كتابة الكود.

---

# مثال من الحياة

كيف تصنع كوبًا من الشاي؟

```
1. أحضر كوبًا.
2. ضع كيس الشاي.
3. اسكب الماء الساخن.
4. انتظر دقيقتين.
5. أضف السكر.
6. قلب جيدًا.
```

هذه خوارزمية.

---

# مثال آخر

تشغيل الكمبيوتر.

```
الضغط على زر التشغيل

↓

انتظار إقلاع النظام

↓

إدخال كلمة المرور

↓

ظهور سطح المكتب
```

---

# لماذا نستخدم الخوارزميات؟

لأنها تساعدنا على:

- تنظيم الأفكار.
- اكتشاف الأخطاء مبكرًا.
- تسهيل كتابة الكود.
- جعل الحل أسهل للفهم.
- تحسين جودة البرامج.

---

# هل كل خوارزمية صحيحة؟

لا.

قد توجد أكثر من خوارزمية لحل نفس المشكلة.

لكن بعضها:

- أسرع.
- أبسط.
- يستهلك ذاكرة أقل.

لذلك يهتم المبرمجون دائمًا بتحسين الخوارزميات.

---

# خصائص الخوارزمية الجيدة

## 1. الوضوح

كل خطوة يجب أن تكون مفهومة.

❌ مثال سيئ:

```
اعمل المطلوب.
```

✅ مثال جيد:

```
اقرأ الرقم الأول.
اقرأ الرقم الثاني.
اجمع الرقمين.
اعرض الناتج.
```

---

## 2. الترتيب

الخطوات يجب أن تكون بالترتيب الصحيح.

لا يمكن عرض النتيجة قبل إجراء العملية.

---

## 3. وجود بداية ونهاية

كل خوارزمية يجب أن تبدأ وتنتهي.

```
Start

↓

خطوات الحل

↓

End
```

---

## 4. عدد محدود من الخطوات

الخوارزمية لا يجب أن تستمر إلى ما لا نهاية.

يجب أن تصل إلى نتيجة.

---

# مثال عملي

المطلوب:

جمع رقمين.

الخوارزمية:

```
Start

↓

اقرأ الرقم الأول.

↓

اقرأ الرقم الثاني.

↓

اجمع الرقمين.

↓

اعرض الناتج.

↓

End
```

---

# مثال آخر

المطلوب:

حساب مساحة مستطيل.

الخوارزمية:

```
Start

↓

اقرأ الطول.

↓

اقرأ العرض.

↓

المساحة = الطول × العرض.

↓

اعرض المساحة.

↓

End
```

---

# مثال ثالث

المطلوب:

معرفة هل الطالب ناجح.

الخوارزمية:

```
Start

↓

اقرأ الدرجة.

↓

إذا كانت الدرجة أكبر من أو تساوي 50

↓

اعرض "ناجح"

وإلا

↓

اعرض "راسب"

↓

End
```

---

# المدخلات والمخرجات

كل خوارزمية تحتوي غالبًا على:

## Inputs

البيانات التي تدخل إلى البرنامج.

مثال:

```
العمر
```

---

## Processing

العمليات التي ينفذها البرنامج.

مثال:

```
العمر = السنة الحالية - سنة الميلاد
```

---

## Outputs

النتيجة النهائية.

مثال:

```
20 سنة
```

---

# مثال شامل

برنامج يحسب متوسط ثلاث درجات.

المدخلات:

```
90

80

70
```

المعالجة:

```
المجموع = 90 + 80 + 70

المتوسط = المجموع ÷ 3
```

المخرجات:

```
80
```

---

# هل نكتب الكود مباشرة؟

لا.

الأفضل دائمًا:

```
فهم المشكلة

↓

كتابة الخوارزمية

↓

كتابة الكود

↓

اختبار البرنامج
```

---

# خوارزمية سيئة

```
احسب الحاجة.

اعرض النتيجة.
```

ما هي الحاجة؟

كيف نحسبها؟

هذه ليست خوارزمية جيدة لأنها غير واضحة.

---

# خوارزمية جيدة

```
Start

↓

اقرأ رقمين.

↓

اجمع الرقمين.

↓

اعرض الناتج.

↓

End
```

كل خطوة واضحة.

---

# أخطاء شائعة

❌ كتابة خطوات ناقصة.

---

❌ تنفيذ العمليات بترتيب خاطئ.

---

❌ نسيان تحديد المدخلات أو المخرجات.

---

❌ الانتقال إلى الكود قبل التأكد من صحة الخوارزمية.

---

# نصائح للمبتدئين

✅ اكتب الحل على الورق أولًا.

✅ استخدم جملًا قصيرة وواضحة.

✅ تأكد أن كل خطوة تؤدي إلى الخطوة التالية.

✅ جرّب تنفيذ الخوارزمية بنفسك قبل تحويلها إلى كود.

---

# ملخص

- الخوارزمية هي خطة لحل المشكلة.
- أي برنامج يبدأ بخوارزمية.
- الخوارزمية الجيدة واضحة ومرتبة ولها بداية ونهاية.
- كتابة الخوارزمية قبل الكود تقلل الأخطاء وتسهل البرمجة.

A) لغة برمجة.

# 🎯 أهداف الدرس

بعد الانتهاء من هذا الدرس ستكون قادرًا على:

- فهم معنى مخططات التدفق (Flowcharts).
- التعرف على الرموز الأساسية المستخدمة في رسمها.
- تحويل أي خوارزمية إلى مخطط تدفق.
- قراءة وفهم أي Flowchart بسيط.
- تصميم Flowcharts لمشكلات برمجية مختلفة.

---

# مقدمة

في الدرس السابق تعلمنا كتابة الحل باستخدام **Pseudocode**.

لكن في المشاريع الكبيرة، يفضل كثير من المبرمجين والمهندسين استخدام **الرسومات** بدلًا من النصوص.

لماذا؟

لأن الرسم يساعد على فهم سير البرنامج بسرعة، ويجعل من السهل اكتشاف الأخطاء قبل كتابة الكود.

---

# ما هو Flowchart؟

الـ **Flowchart** هو رسم بياني يوضح خطوات تنفيذ البرنامج باستخدام رموز وأشكال قياسية متصلة بأسهم.

كل شكل يمثل وظيفة معينة، والأسهم توضح ترتيب التنفيذ.

---

# لماذا نستخدم Flowcharts؟

تساعدنا على:

- فهم المشكلة بسهولة.
- ترتيب خطوات الحل.
- اكتشاف الأخطاء المنطقية.
- شرح الفكرة لفريق العمل.
- تسهيل تحويل الحل إلى كود.

---

# الرموز الأساسية

## 1. Start / End

الشكل:

```
╭────────────╮
│ Start/End  │
╰────────────╯
```

الاستخدام:

بداية أو نهاية البرنامج.

---

## 2. Process

الشكل:

```
┌──────────────┐
│   Process    │
└──────────────┘
```

الاستخدام:

تنفيذ عملية.

مثل:

- جمع رقمين.
- حساب المساحة.
- تخزين قيمة.

---

## 3. Input / Output

الشكل:

```
  ╱────────────╲
 ╱  Input/Output╲
 ╲              ╱
  ╲────────────╱
```

الاستخدام:

- إدخال بيانات.
- عرض النتائج.

---

## 4. Decision

الشكل:

```
      ◇
     / \
    /   \
   Yes  No
```

الاستخدام:

اتخاذ قرار.

مثل:

```
هل الدرجة ≥ 50؟
```

---

## 5. Flow Line

```
↓

↓

↓

```

تمثل اتجاه سير البرنامج.

---

│ Start   │
╰────┬────╯
     │
     ▼

│ Input A      │
╲──────────────╱
     │
     ▼

│ Input B      │

     │
     ▼

     ▼

     ▼

# كيف تقرأ Flowchart؟

ابدأ دائمًا من:

```
Start
```

ثم اتبع الأسهم خطوة بخطوة.

لا تنتقل إلى أي شكل آخر إلا إذا وصل إليه سهم.

---

# قواعد رسم Flowchart

✅ يوجد Start واحد فقط.

---

✅ يوجد End واحد على الأقل.

---

✅ جميع الخطوات متصلة بأسهم.

---

✅ اتجاه التنفيذ يكون من أعلى إلى أسفل قدر الإمكان.

---

✅ كل قرار يجب أن يحتوي على مسارين على الأقل.

مثل:

```
Yes

No
```

---

# الفرق بين Algorithm وPseudocode وFlowchart

| النوع | الشكل |
|--------|--------|
| Algorithm | خطوات مكتوبة |
| Pseudocode | كود وهمي |
| Flowchart | رسم بياني |

جميعها تصف الحل نفسه، لكن بطريقة مختلفة.

---

# أخطاء شائعة

❌ رسم الأسهم في اتجاهات عشوائية.

---

❌ نسيان Start أو End.

---

❌ استخدام شكل خاطئ لعملية الإدخال أو القرار.

---

❌ وجود أكثر من بداية للمخطط.

---

# نصائح احترافية

في المشاريع الصغيرة، قد لا تحتاج إلى رسم Flowchart.

أما في المشاريع الكبيرة أو عند العمل ضمن فريق، فإن مخطط التدفق يساعد على توضيح الفكرة قبل كتابة آلاف الأسطر البرمجية.

---

# ملخص

- Flowchart هو تمثيل رسومي للخوارزمية.
- يستخدم رموزًا قياسية لتمثيل العمليات والقرارات.
- يساعد على فهم الحل قبل كتابة الكود.
- كل مخطط يبدأ بـ Start وينتهي بـ End.
- الأسهم تحدد ترتيب تنفيذ الخطوات.

A) كتابة الكود.

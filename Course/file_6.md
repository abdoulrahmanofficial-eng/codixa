# 🎯 أهداف الدرس

بعد الانتهاء من هذا الدرس ستكون قادرًا على:

- فهم معنى Compiler و Interpreter.
- معرفة الفرق بينهما.
- فهم كيفية تنفيذ البرامج.
- معرفة لماذا تستخدم كل لغة طريقة مختلفة للتنفيذ.
- تصحيح أحد أشهر المفاهيم الخاطئة لدى المبتدئين.

---

# مقدمة

عندما تكتب برنامجًا بلغة مثل Python أو Java أو C++ فإن المعالج لا يستطيع تشغيل هذا الكود مباشرة.

قبل أن يعمل البرنامج يجب تحويله إلى لغة يفهمها الكمبيوتر.

وهنا يظهر دور:

- Compiler
- Interpreter

---

# لماذا نحتاجهما؟

المعالج لا يفهم سوى لغة الآلة (Machine Language).

أما نحن فنكتب بلغات مثل:

- Python
- JavaScript
- Java
- C++

لذلك نحتاج إلى برنامج يقوم بالترجمة بين لغتنا ولغة الكمبيوتر.

---

# ما هو Compiler؟

الـ **Compiler** هو برنامج يحول المشروع بالكامل إلى لغة الآلة قبل تشغيله.

### رحلة التنفيذ

```
Source Code

↓

Compiler

↓

Executable File

↓

تشغيل البرنامج
```

بعد انتهاء الترجمة، يصبح لديك ملف جاهز للتشغيل.

مثل:

```
program.exe
```

---

# مثال

إذا كتبت مشروعًا مكونًا من:

```
1000 سطر
```

فإن الـ Compiler يقرأ جميع الأسطر أولًا.

إذا وجد خطأ في السطر رقم 5 فإنه يتوقف ويخبرك بالمشكلة.

لن يتم إنشاء الملف التنفيذي حتى يتم إصلاح جميع الأخطاء.

---

# مميزات Compiler

✅ سرعة تنفيذ عالية.

✅ أداء ممتاز.

✅ اكتشاف كثير من الأخطاء قبل تشغيل البرنامج.

✅ مناسب للمشاريع الضخمة.

---

# عيوب Compiler

- قد تستغرق عملية الترجمة وقتًا في المشاريع الكبيرة.
- يجب إعادة الترجمة بعد كل تعديل.

---

# أشهر اللغات التي تستخدم Compiler

- C
- C++
- Rust
- Go

---

# ما هو Interpreter؟

الـ **Interpreter** يقرأ الكود وينفذه سطرًا بسطر أثناء تشغيل البرنامج.

### رحلة التنفيذ

```
Source Code

↓

Interpreter

↓

تنفيذ السطر الأول

↓

تنفيذ السطر الثاني

↓

تنفيذ السطر الثالث
```

---

# مثال

لنفترض أن لديك الكود التالي:

```python
print("مرحبًا")

print(5 / 0)

print("تم التنفيذ")
```

سيحدث التالي:

- يطبع "مرحبًا".
- يصل إلى عملية القسمة على صفر فتظهر رسالة خطأ.
- يتوقف البرنامج، ولن يصل إلى السطر الأخير.

---

# مميزات Interpreter

✅ تجربة سريعة أثناء التعلم.

✅ لا يحتاج إلى إنشاء ملف تنفيذي في كل مرة.

✅ مناسب لتجربة الأكواد بسرعة.

---

# عيوب Interpreter

- التنفيذ غالبًا أبطأ من البرامج المترجمة بالكامل.
- تظهر بعض الأخطاء أثناء التشغيل.

---

# أشهر اللغات التي تستخدم Interpreter

- Python
- JavaScript (داخل المتصفح)
- PHP
- Ruby

---

# مقارنة بين Compiler و Interpreter

| Compiler | Interpreter |
|-----------|-------------|
| يترجم البرنامج كاملًا | يترجم سطرًا بسطر |
| التنفيذ أسرع غالبًا | التنفيذ أبطأ قليلًا |
| ينشئ ملفًا تنفيذيًا غالبًا | لا ينشئ ملفًا تنفيذيًا مباشرًا |
| يكتشف كثيرًا من الأخطاء قبل التشغيل | قد تظهر الأخطاء أثناء التشغيل |
| مناسب للبرامج الكبيرة | مناسب للتعلم والتجارب السريعة |

---

# هل Java تستخدم Compiler أم Interpreter؟

Java حالة خاصة.

تمر بعدة مراحل:

```
Java Code

↓

Compiler

↓

Bytecode

↓

JVM

↓

Machine Code

↓

CPU
```

لذلك تجمع بين مزايا الطريقتين.

---

# هل JavaScript مفسرة دائمًا؟

في المتصفحات الحديثة، تستخدم محركات مثل **V8** تقنيات متقدمة لتحسين الأداء، وقد تقوم بترجمة أجزاء من الكود أثناء التشغيل (Just-In-Time Compilation).

لكن كمبتدئ يكفي أن تعرف أنها تُنفذ بطريقة مختلفة عن C++، ولا تحتاج لحفظ هذه التفاصيل الآن.

---

# ماذا يحدث عند الضغط على زر Run؟

عندما تضغط على زر التشغيل في بيئة التطوير:

1. يتم حفظ الكود.
2. تبدأ عملية الترجمة أو التفسير.
3. تتحول التعليمات إلى لغة يفهمها المعالج.
4. يبدأ تنفيذ البرنامج.
5. تظهر النتيجة على الشاشة.

---

# مثال عملي

الكود:

```python
name = "Ali"

print("Hello", name)
```

الخطوات:

```
كتابة الكود

↓

Interpreter

↓

Machine Instructions

↓

CPU

↓

الشاشة
```

الناتج:

```
Hello Ali
```

---

# أخطاء شائعة

❌ الاعتقاد أن Python تُنفذ مباشرة بواسطة المعالج.

الصحيح:

تمر بمراحل قبل أن تصل إلى تعليمات يمكن للمعالج تنفيذها.

---

❌ الاعتقاد أن Compiler دائمًا أفضل.

الصحيح:

يعتمد ذلك على نوع المشروع واحتياجاته.

---

❌ الاعتقاد أن جميع لغات البرمجة تعمل بالطريقة نفسها.

الصحيح:

لكل لغة طريقة تنفيذ تناسب أهدافها.

---

# ملخص الدرس

- المعالج لا يفهم إلا لغة الآلة.
- Compiler يترجم البرنامج بالكامل قبل تشغيله.
- Interpreter ينفذ الكود أثناء التشغيل.
- لكل طريقة مزايا وعيوب.
- اختيار اللغة يعتمد على المجال، وليس على طريقة التنفيذ فقط.
